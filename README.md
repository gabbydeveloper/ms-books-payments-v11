# Payments

## 📌 Descripción

Este proyecto es un microservicio encargado de gestionar **pagos y ventas** dentro de una arquitectura basada en microservicios.

Su responsabilidad principal es:

- Registrar ventas
- Procesar pagos
- Validar transacciones
- Gestionar estados de pago

---

## 🧱 Responsabilidades

- ✅ Crear y registrar ventas
- 💳 Procesar pagos
- 🔄 Actualizar estado de ventas
- 📄 Consultar historial de ventas
- 🔐 Validar integridad de transacciones
- ✅ Además maneja usuarios ingresados

---

## 🏗 Arquitectura

El microservicio sigue principios de:

- Arquitectura basada en microservicios
- Separación de responsabilidades
- API REST
- Comunicación síncrona (HTTP) y/o asíncrona (eventos, si aplica)
